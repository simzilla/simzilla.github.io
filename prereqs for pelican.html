Pelican Prerequisites

Posted on Fri 08 July 2016

Things to do before getting started

I've put very little thought into this and have decided on an approach to take with this thing. I'm going to write about my journey into Pelican as though I am Rufus and you are Bill S. Preston or Ted 'Theodore' Logan...whichever its up to you...depending on which lead guitar you like to play I guess.

Anyway, I am going to keep this super simple and assume that nothing on your machine is prepared for the Pelican journey. Something that I would have liked when I was getting started...

Just as a disclaimer, alot of this is available at the Pelican homepage. I am going to try and break it down really simple but if I fail or leave something out you can always refer to that site.

I'm going to cover the software that you HAVE to install to make Pelican work and the stuff I recommend you install in order to make this all a little easier.

Foundation

Pelican uses Python to create the web content that you will store on a server somewhere so in order to use Pelican you will need to make sure you have Python installed on your computer. I personally use a Mac so Python was on my computer right out of the box (Version 2.7, but I have since upgraded to Version 3). I'm not sure about PCs running windows. The easiest way to check is to open up a terminal and type

python


If this works your prompt will change from a ">" or "#" to a ">>>" and you will see some information about your Python version.

This is where I ran into the most problems. Its possible for you to have to versions of python installed on your machine and have conflicts when you try and use one or the other or install packages, such as Pelican. This typically occurs due to Path variable errors. I'm not going to get into this now but just be careful. If you are like me, an intelligent Mac owner, use a package manager like Homebrew to install and update Python.

Pelican

If you installed the newest version of Python or are using a package manager you should have pip, a python specific package manager, installed on your computer. Pip will make the installation of Pelican really easy. Give this a try:

pip install pelican


This will install the following packages into you environment: feedgenerator, jinja2, pygments, docutils, pytz, blinker, unidecode. You should get a success message once the installation is complete.

That's really it.

Recommendations

As I've said before, Pelican is very lightweight so you have many options when it comes to hosting. You can setup a server if you so desire, or if you want to do it the way I found the most convenient try Github pages. Github has alot of features, but in this case its an online repository where you can store your html so that you can display it on Github pages.

Github Setup

First you are going to want to head over to Github and create an account if you don't already have one. I find it alot easier to manage Github repositories (uploading files and managing updates) with a GUI. On my Mac I use Github Desktop and it makes managing your blog really simple. For a really easy to follow guide head to the Github pages website.

Basically, all you have to do is create Github repository using the naming convention specified in the guide. That repository shares the same name as your new blog. When you run Pelican the output goes into a folder that you subsequently push into this repository. I'll go over it again but its super simple.

Markdown Editor

Another piece of software I recommend is Mou. You can use any text/code editor you want but I really like Mou. It has several really useful features. For one, as you write your blog and input Markdown syntax, one of the formats that Pelican uses to create blog posts, you get a real time preview in a split screen. It is pretty awesome and makes writing posts really straight forward. It even responds to html, so as I throw some code into a post to center or size a picture I get my realtime preview and have a good idea of how my output will look.

Mou is currently in beta, and I am pretty sure it only works on OS X.

MISC...

I've personally found a few other things useful. I use Chrome as my browser for web development. I find that the developer tools work better (cache isn't so finicky and I can dive into html if I need to). I've also been trying out iTerm (a terminal interface) and so far I like it but its definitely not essential.
